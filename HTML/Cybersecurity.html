Cybersecurity

Cybersecurity (ความมั่นคง ความปลอดภัยทางไซเบอร์)

  • คือ การปกป้องระบบ เครือข่าย อุปกรณ์ และข้อมูลจากการเข้าถึง การเปลี่ยนแปลง หรือการทำลายโดยไม่ได้รับอนุญาต หรือจากภัยคุกคามทางไซเบอร์อื่นๆ

เป้าหมายหลักของ Cybersecurity (CIA)

  • Confidentiality (ความลับ) ป้องกันไม่ให้บุคคลที่ไม่ได้รับอนุญาตเข้าถึงข้อมูล
  • Integrity (ความถูกต้อง) รักษาให้ข้อมูลไม่ถูกแก้ไขหรือเปลี่ยนแปลงโดยไม่ได้รับอนุญาต
  • Availability (ความพร้อมใช้งาน) ทำให้ระบบและข้อมูลสามารถใช้งานได้ตลอดเวลาเมื่อจำเป็น

ตัวอย่างภัยคุกคามทางไซเบอร์

คือ โปรแกรมที่เป็นอันตราย เช่น ไวรัส, เวิร์ม, โทรจัน, แรนซัมแวร์, และสปายแวร์

คือ การหลอกลวงเพื่อขโมยข้อมูลส่วนตัว เช่น รหัสผ่าน, เลขที่บัตรเครดิต

คือ การโจมตีที่ทำให้ระบบหรือเครือข่ายไม่สามารถใช้งานได้

คือ การหลอกลวงทางอีเมลที่ผู้โจมตีแอบอ้างเป็นบุคคลสำคัญเพื่อหลอกให้เหยื่อโอนเงิน หรือเปิดเผยข้อมูล

คือ การหลอกลวงด้วยจิตวิทยาเพื่อให้เหยื่อยอมเปิดเผยข้อมูลหรือทำบางอย่างโดยสมัครใจ

วิธีแก้ปัญหา

  • สแกนคอมพิวเตอร์ด้วยโปรแกรม Antivirus ทันที
  • เข้า Safe Mode แล้วลบไฟล์มัลแวร์ด้วยเครื่องมือเฉพาะ (เช่น Malwarebytes)
  • กู้คืนระบบด้วย System Restore (ถ้ามีจุดคืนค่าก่อนติดมัลแวร์)
  • เปลี่ยนรหัสผ่านทั้งหมดหลังจากทำความสะอาดระบบ
  • ตรวจสอบพฤติกรรมผิดปกติหลังการลบ เช่น การใช้งาน CPU สูงผิดปกติ
  • หากเผลอกรอกข้อมูลไปแล้ว → รีบ เปลี่ยนรหัสผ่านทันที
  • แจ้ง หน่วยงานที่เกี่ยวข้อง เช่น ธนาคาร, อีเมล, บริษัท ฯลฯ
  • ตรวจสอบว่าบัญชีถูกเข้าถึงหรือเปลี่ยนแปลงอะไรหรือไม่
  • สแกนอุปกรณ์หาไฟล์ที่อาจติดมัลแวร์จากอีเมลนั้น
  • แจ้งผู้ให้บริการโฮสติ้งหรือ CDN ให้ช่วยบล็อกทราฟฟิกที่เป็นอันตราย
  • ใช้ระบบ Redirect ชั่วคราว หรือเปลี่ยน IP (ถ้าเป็นไปได้)
  • วิเคราะห์ Log เพื่อระบุแหล่งที่มา และปิดพอร์ตที่ไม่จำเป็น
  • กู้คืนระบบหลังจากการโจมตีหยุดลง เช่น ตรวจสอบฐานข้อมูลหรือไฟล์เสียหาย
  • เปลี่ยนรหัสผ่านและ ตรวจสอบการล็อกอินย้อนหลัง ว่ามีเข้าจากต่างประเทศหรือไม่
  • แจ้ง IT Security เพื่อปิดช่องโหว่และติดตามการเข้าถึง
  • ตรวจสอบว่ามีการส่งอีเมลปลอมจากบัญชีจริงหรือไม่
  • แจ้งธนาคารหากมีการโอนเงินผิดบัญชี → ขออายัดธุรกรรมทันที
  • ตรวจสอบข้อมูลที่ถูกเปิดเผยแล้ว และประเมินความเสียหาย
  • เปลี่ยนรหัสผ่าน บัตรเข้าระบบ หรือข้อมูลยืนยันตัวตน
  • แจ้งเจ้าหน้าที่ความปลอดภัย หรือหัวหน้าฝ่ายทันที
  • หากเกิดการรั่วไหลข้อมูลลูกค้า → แจ้งลูกค้าและหน่วยงานรัฐตามกฎหมาย

วิธีป้องกัน

  • ติดตั้งโปรแกรม Antivirus/Anti-malware ที่น่าเชื่อถือ และอัปเดตสม่ำเสมอ
  • หลีกเลี่ยงการดาวน์โหลดไฟล์จากเว็บไซต์หรืออีเมลที่ไม่น่าเชื่อถือ
  • ห้ามเสียบ USB หรืออุปกรณ์ที่ไม่รู้แหล่งที่มา
  • อัปเดตระบบปฏิบัติการและแอปพลิเคชันให้เป็นเวอร์ชันล่าสุด
  • อย่าคลิกลิงก์หรือเปิดไฟล์แนบจากอีเมลที่น่าสงสัย
  • ตรวจสอบ URL ให้แน่ใจว่าเป็นของจริง (ดู HTTPS และชื่อโดเมน)
  • ซ่อน IP Address ของเซิร์ฟเวอร์หลัก
  • ใช้บริการ DDoS Protection เช่น Cloudflare, AWS Shield
  • ใช้ 2FA กับบัญชีอีเมลองค์กร
  • จำกัดสิทธิ์การเข้าถึงอีเมล และตั้งค่าความปลอดภัยให้สูง
  • ใช้นโยบาย ยืนยันสองขั้นตอนก่อนการโอนเงิน
  • อบรมพนักงานให้รู้จักเทคนิคการหลอกลวง เช่น แอบอ้าง, กดดัน, เสนอของฟรี
  • อย่าเปิดเผยข้อมูลส่วนตัว/ข้อมูลบริษัทในที่สาธารณะหรือโซเชียลมีเดีย
  • สร้างขั้นตอนยืนยันตัวตนก่อนเปิดเผยข้อมูลสำคัญ
